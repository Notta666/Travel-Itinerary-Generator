# Release Notes - v1.2.1

AI 驱动智能旅行攻略生成器 `v1.2.1` 正式发布！本次更新主要修复了在多城市联玩旅行规划时景点的“跨城定位偏离”与“配图错配”的顽疾，并引入了智能城市分区定位校验机制。

## 🔍 问题背景与诊断

在之前的版本中，如果生成多城市（例如“广州,顺德,珠海,澳门”）行程，当进入 Step 3 地理编码和 Step 9 配图阶段时，由于缺乏各景点/餐厅的具体所属城市信息，系统会盲目地以前缀顺序遍历来请求高德 API（如优先查询 `"广州"` 前缀）。
这导致“顺德清晖园”因为被拼接为“广州清晖园”而误匹配到了广州越秀区的“清晖苑茗食府”（位于亚洲国际大酒店），不仅坐标偏离近 40 公里，其配图也被错误地替换成了酒店餐厅；“珠海情侣路”也被偏置到广州花都的“情侣主题酒店”。

## 🛠️ 修复与增强内容

为了彻底解决上述错配和跨城偏差问题，本版本进行了如下重构：

### 1. 景点与美食城市归属判定
- **LLM 双向智能提取**：Step 2 小红书提取的 POI 与餐厅 JSON 结构中增加了 `city` 字段，要求 LLM 智能分类输出各条目所属具体城市。
- **缺失城市归属快速校准**：在 Step 3（景点编码）和 Step 4（美食编码）前，如果检测到 POI 缺乏城市归属（如用户在命令行手动传入的 `--pois` 或 XHS 未联网情况下的默认景点），系统会启动一次极速 LLM 批量映射机制，根据目的地城市列表进行智能分区分类。

### 2. 智能城市约束地理编码
- **城市限制精准匹配**：Step 3 和 Step 4 升级了地理编码算法，限制并在对应城市及其同源城市（如顺德→佛山）内搜索，彻底隔绝异地同名 POI 带来的干扰。
- **防止同名店漂移**：美食批量地理编码时剔除了带逗号的目的地字符串，改由为每个美食在其专属城市下并行高精度编码。

### 3. 出图精准度恢复
- 随着定位的校准，Step 9 图片拉取时拼接的搜索前缀也恢复为正确的城+景（如 `"顺德清晖园"`、`"珠海情侣路"`），高德和搜索引擎将准确下载真实本尊的美图，不会再被错误图片污染缓存。
